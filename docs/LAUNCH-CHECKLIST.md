# Launch Checklist (R1–R4, L1, L4, L5)

Actionable steps for community launch. See `docs/LAUNCH-AND-COMMUNITY.md` for full context.

---

## R1 / L1: Publish Manifesto

**Task:** Post AGNI intro to HN, Reddit, Dev.to.

**One-line pitch:** AGNI compiles YAML lessons into tiny, offline-first HTML bundles that run on 10-year-old phones and use sensors (accelerometer, etc.) for interactive learning.

**Steps:**
1. [ ] **HN (Show HN):** Post to https://news.ycombinator.com/submit with title like "Show HN: AGNI – Offline YAML lessons for sensor-rich learning on old phones"
2. [ ] **Reddit:** Post to r/opensource, r/selfhosted, or r/educationaltechnology. Include link to repo, MANIFESTO.md, and README.
3. [ ] **Dev.to:** Publish a short post with manifesto summary + link. Tag: education, open-source, offline-first.

**Links to include:** https://github.com/NewmanB1/AGNI, README, CONTRIBUTING.md, MANIFESTO.md.

---

## R2 / L4: Triage Issues

**Task:** Review open GitHub issues so contributors know what's in scope.

**Steps:**
1. [ ] Go to https://github.com/NewmanB1/AGNI/issues
2. [ ] For each open issue: Close if obsolete, add labels, assign milestone if applicable
3. [ ] Add "good first issue" to small, well-scoped tasks
4. [ ] **Ongoing:** Triage new issues weekly or per-release

---

## R3: Label "Good First Issues"

**Task:** Apply `good first issue` (and area labels) so contributors can filter.

**Steps:**
1. [ ] Ensure `.github/labels.yml` or GitHub repo Labels include: `good first issue`, `translation`, `documentation`, `area: hub`, `area: runtime`, etc.
2. [ ] Apply `good first issue` to: translate-one-lesson, doc typo, small compiler/runtime fix
3. [ ] Point new contributors to `docs/tutorials/fork-and-translate-lesson.md` for translation

---

## R4 / L5: Record Video

**Task:** Record 5–10 min "How to Fork and Translate a Lesson" video.

**Steps:**
1. [ ] Use `docs/tutorials/fork-and-translate-lesson.md` as script
2. [ ] Record screen + voice (OBS, Loom, or similar)
3. [ ] Upload to YouTube / Vimeo / Dev.to
4. [ ] Add link to README or CONTRIBUTING.md
