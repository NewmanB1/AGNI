# Community & Launch Checklist

Templates and checklists for roadmap items R1–R4, L1, L4, L5. These are **process/outreach tasks** — no code dependencies.

---

## R1 / L1: Publish Manifesto to HN, Reddit, Dev.to

**When:** Day 76–80 (or when ready for public launch)

**Checklist:**
- [ ] Draft intro post (2–3 paragraphs): what AGNI is, why offline/sensor learning, OLS standard
- [ ] Prepare demo link (e.g. `https://your-hub/lessons/gravity` or portal)
- [ ] **Hacker News:** Submit to [Show HN](https://news.ycombinator.com/submit) — "Show HN: AGNI – compile YAML lessons to offline HTML for Raspberry Pi / Android"
- [ ] **Reddit:** r/selfhosted, r/opensource, r/education, r/raspberry_pi — follow each sub's self-promo rules
- [ ] **Dev.to:** Publish as article with code snippets, architecture diagram
- [ ] Respond to comments within 24–48h

**Template (HN title):**
> Show HN: AGNI – Open Lesson Standard compiler for offline, sensor-rich learning on Pi/Android

---

## R2: Triage Issues

**When:** Day 81–85 (or weekly)

**Checklist:**
- [ ] Review all open issues
- [ ] Close duplicates, mark WONTFIX with reason
- [ ] Add labels: bug, enhancement, documentation, good-first-issue
- [ ] Assign priority (P0/P1/P2/P3) where applicable
- [ ] Link related PRs to issues

---

## R3: Label "Good First Issues"

**When:** Day 81–85

**Checklist:**
- [ ] Identify issues suitable for new contributors (clear scope, low risk)
- [ ] Add `good-first-issue` label
- [ ] Ensure CONTRIBUTING.md explains how to pick one
- [ ] Add "Help wanted" section to README if desired

---

## R4 / L5: Record 5–10 min Tutorial Video

**When:** Day 86–90 (optional)

**Checklist:**
- [ ] Script: install → compile a lesson → run hub → open in browser
- [ ] Show: fork lesson, edit YAML, recompile
- [ ] Record (OBS, Loom, or similar)
- [ ] Upload to YouTube / peertube
- [ ] Add link to README and docs

---

## L4: Triage Issues Regularly

**When:** Weekly or per sprint

**Checklist:**
- [ ] New issues: triage within 3–5 days
- [ ] Stale issues: close or update
- [ ] Good-first-issue: ensure at least 2–3 available
